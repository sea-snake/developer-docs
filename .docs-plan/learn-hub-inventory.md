## Linking Strategy

For each docs page that has Learn Hub counterparts:

1. **Concepts pages**: Add a "Deep dive" or "Learn more" section at the bottom with direct links to specific Learn Hub articles (not just the homepage)
2. **Guide pages**: Where Learn Hub explains the "why" behind a feature, add inline links like "To understand how consensus achieves this, see [Learn Hub: Consensus](url)"
3. **Reference pages**: Link to Learn Hub for protocol context where relevant

Our docs focus on **developer-relevant** explanations (what does this mean for my app?). Learn Hub provides **protocol-level** depth (how does the protocol implement this?).

### End-user content (19 articles) — DO NOT link from developer docs
The "How can I use ICP?" category is mostly NNS dapp end-user guides (staking, sending tokens, logging in). These are not developer-facing and should not be linked from developer docs.
